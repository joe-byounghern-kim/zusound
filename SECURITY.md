# Security Policy

## Supported Versions

Security fixes are applied to the latest released version.

| Version        | Supported |
| -------------- | --------- |
| Latest release | Yes       |
| Older releases | No        |

## Reporting a Vulnerability

Please do not open public issues for security reports.

Use one of the following private channels:

1. GitHub Security Advisories for this repository (preferred)
2. Private maintainer contact channel if available

Include:

- A clear description of the issue
- Reproduction steps or proof-of-concept
- Impact assessment
- Suggested mitigation (optional)

## Response Expectations

- Initial triage target: within 3 business days
- Follow-up cadence: periodic updates until resolution

## Disclosure Policy

- We follow coordinated disclosure.
- Please allow time for a fix before public disclosure.

## Scope Notes

- This is a client-side debugging middleware library (no hosted service).
- Security work focuses on dependency integrity, safe defaults, and browser-side stability.
